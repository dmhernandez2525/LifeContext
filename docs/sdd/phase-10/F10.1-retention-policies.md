# F10.1 Retention Policies - SDD

## Overview
Added configurable data retention policies with per-category retention periods, auto-cleanup scheduling, and purge tracking.

## Files Created

### `apps/web/src/lib/retentionPolicies.ts`
- 6 data categories: journal, brainDump, questions, browserHistory, cookies, voiceDocs
- 7 retention periods: forever, 30d, 90d, 180d, 1y, 2y, 5y
- `getCutoffDate()` - Calculates date threshold for a retention period
- `isPurgeDue()` - Checks if cleanup should run based on interval
- `countExpiredItems()` - Counts items older than cutoff for a category
- `getPurgeSummary()` - Aggregates purge preview across all policies
- Default policies: journal/brainDump/questions = forever, browserHistory = 1y, cookies = 90d, voiceDocs = 180d

### `apps/web/src/store/retention-store.ts`
- Zustand persisted store for retention configuration
- Actions: updatePolicy, togglePolicy, setAutoDelete, setPurgeInterval, recordPurge, resetToDefaults

### `apps/web/src/components/settings/RetentionSettings.tsx`
- Per-category policy rows with checkbox enable and period selector
- Auto-cleanup toggle with configurable interval (daily/weekly/biweekly/monthly)
- Last cleanup timestamp with "Run now" button
- Reset to defaults button
- Summary text showing count of active time-based policies
