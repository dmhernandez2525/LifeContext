# F10.3 Journey Import - SDD

## Overview
Added Journey app journal import parser with mood mapping, location/weather extraction, and integration into the unified ImportPage.

## Files Created

### `apps/web/src/lib/importers/journey.ts`
- `parseJourneyExport(json)` - Parses Journey JSON format (supports both `{ entries: [...] }` and flat array)
- `isJourneyFormat(json)` - Validates file structure
- `mapMood(mood)` - Maps Journey's 1-10 numeric mood scale to JournalMood enum
- Location formatting from address, label, or lat/lon coordinates
- Weather description and temperature extraction

### `apps/web/src/components/import/JourneyImport.tsx`
- Same four-phase flow as DayOneImport: upload, preview, complete, error
- Drag-and-drop file zone with JSON file picker
- Journey-specific export instructions in upload prompt
- Reuses shared ImportPreview component

## Files Modified

### `apps/web/src/lib/importers/index.ts`
- Added Journey parser exports

### `apps/web/src/components/import/index.ts`
- Added JourneyImport export

### `apps/web/src/pages/ImportPage.tsx`
- Activated Journey tab (was "Coming Soon")
- Wired JourneyImport component to import handler

## Architecture Notes
- Journey mood scale (1-10) mapped to 5-level JournalMood via range buckets
- Handles Journey's two export formats: object with entries array and flat array
- Shares importHandler and ImportPreview with DayOne import (F10.2)
